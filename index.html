Intro to Greensock (GSAP)


  • Create performant animations
  • Handles SVG animations
  • It can be used safely (back to IE6!) to create animations without jank
  • Greensock /GSAP is the company
  • TweenMax / TimelineMax is the library

TweenMax and TimelineMax


  • GSAP lets you choose from four libraries—TweenMax has more features than TweenLite; TimelineMax has more features than TimelineLite.
  • We'll optimize for learning and use TweenMax since it includes almost everything we'll need (including TimelineMax).
  • Installing GSAP is as simple as including the script. See CDN links

What's a Tween?


A tween is a single movement in an animation. In GSAP, a tween has the following syntax: TweenMax.method(element, duration, vars)

  • method refers to the GSAP method you’ll like to tween with:
    1. set
    2. from
    3. to
  • element is the element you want to animate. If you want to create tweens for multiple elements at the same time, you can pass in an array of elements to element
  • duration is the duration of your tween. It is an integer in seconds without the 's' suffix!
  • vars is an object of the properties you want to animate

GSAP Methods


GSAP lets you animate with many methods. When you’re new, the ones you’ll use most are: 1) set, 2) from, 3) to

  • setsets the vars object to the specified values without creating an animation.
  • from animates the element from the values you set in vars to their current values.
  • to animates the element from their current values to the values you set in vars

GSAP Vars


vars is an object that lets you specify the properties you want to animate.
You can specify any CSS property by switching the CSS naming format into camel case.

  • For example, if you want to animate the font-size property,
    you change font-size into fontSize. 
    const vars = {
    fontSize: '20px',
    backgroundColor: '#40eefa'
  }
  • GSAP also lets you animate CSS transforms. The syntax here is slightly different though:
    1. For translateX, you use x
    2. For translateY, you use y
    3. For rotate, you use rotation

    const vars = {
    x: 200, // Translates 200px from left to right
    y: -200 // Translates 200px from bottom to top
    rotation: 90 // Rotates 90 degrees
  }

Even though GSAP is super fast, you can’t stop jank from happening if you animate CSS properties other than transform and opacity

Tweening .to

Move the box from the left to the right over 2 seconds. 


  const box = document.querySelector('.box')
  TweenMax.to(box, 2, {x: 400 })

Timing Functions

GSAP provides you with many easing variables like Power0 to Power4 


      const vars = {
        ease: Power0.easeOut
      }

In addition, GSAP gives you other easing variables: Bounce, Elastic, etc. that are more powerful than to cubic-bezier

Easing Visualizer

you can use the easing visualizer that GSAP has created

Timing functions

Experiment with different eases 


    const box = document.querySelector('.box')
    TweenMax.from(box, 2, {
      y: 1000,
      ease: Power0.easeOut
    })
                    

    const ball = document.querySelector('.ball');
    
    TweenMax.to(ball, 3, {
      x: 600,
      rotation: 720,  
      transformOrigin: '50% 50%',
      ease: 'Power1.easeOut'
    });    
      
    TweenMax.to(ball, 2, {
      y: 400, 
      ease: 'Bounce.easeOut'
    });

Multiple Tweens at the same time
